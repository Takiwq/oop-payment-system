## 📋 Опис

Міні-банк з різними типами рахунків, транзакціями та переказами.  
Реалізовано систему з абстрактними класами, наслідуванням, магічними методами та unit-тестами.

---

## 🗂 Структура проєкту

| Файл | Опис |
|------|------|
| `models.py` | Базові моделі: `Owner`, `Transaction`, Enum-типи |
| `accounts.py` | `Account` (ABC), `SavingsAccount`, `CreditAccount`, `CurrencyAccount` |
| `payment_system.py` | `PaymentSystem` — реєстр рахунків та перекази |
| `test.py` | 59 unit-тестів (unittest) |
| `main.py` | Демонстрація роботи |

---

## 🏗 Архітектура класів

```
ABC
 └── Account  (абстрактний)
      ├── SavingsAccount       ← ощадний, відсотки, не йде в мінус
      │    └── CurrencyAccount ← валютний (USD/EUR/GBP/PLN), конвертація
      └── CreditAccount        ← кредитний, ліміт, комісія
```

Глибина наслідування: **4 рівні** (ABC → Account → SavingsAccount → CurrencyAccount)

---

## ✨ Функціонал

- 💰 Поповнення та зняття коштів з валідацією
- 🔄 Перекази між рахунками через `PaymentSystem`
- 📈 Автоматичне нарахування відсотків (`SavingsAccount`)
- 💸 Нарахування комісії на від'ємний залишок (`CreditAccount`)
- 💱 Валютні рахунки з конвертацією за курсом (`CurrencyAccount`)
- 🔒 Блокування та розблокування рахунків
- 📄 Виписка по рахунку (`get_statement`)
- 🔍 Пошук рахунків за ІПН власника
